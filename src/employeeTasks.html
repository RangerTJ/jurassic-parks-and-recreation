Employee Tasks

Add New Employee Task

If you would like to add a new Employee Task to the database. Enter values for its attributes below and click the "Add Task" button. Employee tasks are entities that handle the many-to-many relationship between a given Employee and an open Task Assignment. Each entity represents a chunk of work contributed by a specific employee towards the completion of the matching task. Many employees can contribute to the same task, and a single employee can contribute to the same task multiple times (for different hourly contributions at different time intervals).

Alternative Design Proposal Musings

Alternate Solution: Input field for delete based on target criteria (like ID or unique attribute). Add ID field to add new parks, and branch logic so that if it's NULL, a new one is added, and if it's not NULL, then create a new one? Having delete be a lookup saves needing deletes for each row. Alternate update would be update lookup too, but then need new update page for each entity for updating instead of adding. Would then be add/delete page and a different modify page. Add/delete could be the main page with the report/summary table of the general entity query. Alternately, could make that a lookup too, instead. Launcher page could be a welcome and have buttons/inputs/drop-downs that launch the "popular" custom queries. Might be a good idea to use drop downs in place of inputs when it's by an ID, so the drop-down could display the ID + an additional attribute to clarify what each ID is (for dinos/tasks and things that don't have unique non-id identifiers).
